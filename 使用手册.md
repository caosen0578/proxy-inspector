# 资金同业代码解析工具 · 使用手册

抓取你的 AI 编码数据并自动上送,统计代码生成率。上送配置已由管理员配好,你只需**装证书 + 设代理 + 启动**。

> 前提:本机装了 **Node.js ≥ 20**(命令行 `node -v` 查看;低于 20 或没装,找管理员装/升级)。
> ⚠️ **务必 Node 20 及以上**——低版本会导致 HTTPS 抓取偶发失败(报 3003 Cannot connect: Headers Timeout)。全程不需要管理员权限。

---

## 启动

**方式一:一键脚本(推荐)**
- **Windows**:双击 **`start.cmd`**
- **统信 UOS / Linux**:终端执行 **`bash start.sh`**

**方式二:手动命令**(脚本被拦截、或想自定义端口时用)
在工具目录下打开命令行/终端,执行:
```
node src/index.js
```
自定义端口(可选):
- Windows:`set PROXY_PORT=28899 && set UI_PORT=28900 && node src/index.js`
- Linux:`PROXY_PORT=28899 UI_PORT=28900 node src/index.js`

启动后保持窗口开着,会显示:
```
Proxy: http://127.0.0.1:28899    UI: http://127.0.0.1:28900
```

---

## 装根证书(抓 HTTPS 必需,只做一次)

1. 浏览器打开 `http://127.0.0.1:28900`,随便访问一次任意 https 网站(生成证书)。
2. 点界面上的 **「⬇ 下载根证书」**,得到 **`.cer`** 文件。
3. 双击 `.cer` → 安装 → 存储位置选 **「当前用户」**(不需要管理员)→ 放入 **「受信任的根证书颁发机构」** → 完成。
4. 重启浏览器 / IDE。

---

## 设代理(把 AI 工具指向 `127.0.0.1:28899`)

按你用的工具选一种:

- **CodeBuddy 独立 IDE**:打开**设置**,搜索框输入 **「代理」**,在代理设置项里填 `127.0.0.1` 端口 `28899`。
- **IDEA + CodeBuddy 插件**:打开**设置(Settings/Preferences)**,搜索框输入 **「codebuddy」**,找到 CodeBuddy 的 **「网络代理地址」** 项,填 `127.0.0.1:28899`。

填好保存,正常用 AI 编码即可。

## 界面怎么看(`http://127.0.0.1:28900`)

日常你基本不用操作,工具会自动采集上送。界面上你能用到的:

| 位置 / 按钮 | 作用 |
|------|------|
| 顶部 **「⬇ 下载根证书」** | 装 HTTPS 证书用(见上一步) |
| 顶部 **「上送」开 / 关** | 显示上送是否开启(应为"开";灰了找管理员) |
| **左侧抓包列表** | 实时显示抓到的 AI 请求,**点一条**看报文详情;上方可按 URL/类型筛选 |
| **「上送队列」标签** | 看上送情况:绿色 **「已送达」= 成功** ✅;红色失败可点该行 **「重试」** |
| 「上送队列」→ **「补送历史报文」** | 上送开晚了漏了几条?点它勾选补送 |

- 看不到 **「上送设置 / 字段映射」** 两个标签是**正常的**——那是管理员配置项,已配好,你无需也无法修改。
- 右上角 **「UM xxx」** 是你的统计标识(自动取系统用户名),不对找管理员改。

## 让它一直开着(推荐一次装好)

工具关了就不采集。装开机自启,以后登录自动后台运行:

- **Windows**:双击 `scripts\autostart\install-autostart-win.cmd`
- **Linux**:`bash scripts/autostart/install-autostart.sh`

---

## 停止

前台窗口按 **Ctrl+C**;后台自启用同目录的 `uninstall-autostart-*` 卸载。

有问题联系管理员。

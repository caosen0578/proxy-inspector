示例案例（点击「填入表单」参考）

① Mock 一个接口返回固定 JSON
后端没就绪时用假数据联调。命中的请求不转发，直接返回你设的 body。
*/v1/chat/completions → Mock 200 {"reply":"hello"}
② 给请求注入鉴权头
请求正常转发，额外加上 Header（如临时 token、调试标记）。
*api.example.com* → 加请求头 {"Authorization":"Bearer xxx"}
③ 给响应加 CORS 头
本地跨域调试时，给响应补上 Access-Control-Allow-Origin。
* → 加响应头 {"Access-Control-Allow-Origin":"*"}
④ 阻断遥测/广告请求
命中的请求直接返回 403，不放行。可屏蔽埋点/统计/广告。
*telemetry* → 阻断（403）

使用说明

  • 用途：在代理转发时拦截/改写请求，与「上送埋点」相互独立，按需使用。
  • 匹配：URL 模式支持子串、* 通配符、/正则/；可限定 HTTP 方法。按顺序匹配，命中第一条即生效。
  • 三种动作：
    • Mock 响应 —— 不转发真实服务器，直接返回设定的状态码 / Content-Type / Body。
    • 修改请求/响应头 —— 正常转发，额外追加（覆盖）Header，填 JSON。
    • 阻断请求 —— 直接返回 403。
  • 每条规则可单独启用/禁用（左侧开关），无需删除即可临时关闭。
  • 规则保存后立即生效，对随后经过代理（127.0.0.1:8899）的流量起作用。

每行支持：子串包含（如 /v1/chat）、通配符（如 *glm*）、正则（如 /embeddings|completion/）。命中任意一行即上送。

匹配的请求走快速通道：不强制 gzip、关闭 Nagle、转发零处理，尽量不破坏 IDE 的行内代码补全回写。采集与上送照常。默认 */completions*（覆盖代码补全/对话两个接口）。

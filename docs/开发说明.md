# 资金同业代码解析工具 - 项目说明文档

> 最后更新：2026-06-17

---

## 一、项目概述

- **工具名称**：资金同业代码解析工具（可在「上送设置」里修改）
- **工作目录**：`D:\code\proxy-inspector`
- **本质**：Node.js HTTP/HTTPS MITM 代理 + Web 管理面板
- **用途**：拦截 CodeBuddy IDE 插件与 AI 服务之间的流量，提取 AI 产出的代码，自动上送至行内用户行为埋点接口，统计个人代码生成率
- **面板地址**：http://127.0.0.1:28900
- **启动方式**：`start.cmd`（Windows）；打包：`pack.cmd`

## 三、核心模块

### `src/reporter-mapping.js` — 报文字段映射 & 代码提取

**两个 CodeBuddy 接口的提取规则：**

#### 接口一：`/v2/completions`（行内代码补全 CodeCompletion）

SSE 格式：每个 chunk 用 `choices[0].text`（裸代码，无 ``` 围栏）

| 埋点字段 | 提取规则 |
|---------|---------|
| result / acceptResult | `isCompletionStyle`=true → `sseJoinContent` 拼接所有 `choices[0].text`（全量裸代码） |
| codeLines | `result.split('\n').length` |
| codeSize | `Buffer.byteLength(result, 'utf8')` |
| modelName | 任意 chunk 的 `model` 字段 |
| promptTokens | 最后含 `usage` 的 chunk 的 `usage.prompt_tokens` |
| completionTokens | 最后含 `usage` 的 chunk 的 `usage.completion_tokens` |
| finishReason | 最后含 `finish_reason` 的 chunk 的 `choices[0].finish_reason` |
| language | 请求体 `extra.language` |
| filePath | 请求体 `extra.file_name` |
| repository | 请求体 `extra.repo_name` |

#### 接口二：`/v2/chat/completions`（Craft/Agent 对话模式）

SSE 格式：`choices[0].delta.content`（说明文字）+ 可能含 `delta.tool_calls`（工具调用）

`extractGeneratedCode` 优先级（三选一）：

1. **写入型工具调用**（最高优先）：遍历 SSE 中所有 `delta.tool_calls`，拼接 `function.arguments` → JSON.parse → 取 `new_str ?? content ?? code ?? newText`。
   - `write_to_file.content` = 完整文件代码
   - `replace_in_file.new_str` = 替换后代码片段
   - 只读工具（`read_file` / `read_lints` / `search`）无这些字段 → 自动跳过
   - 多个写入工具调用用 `\n\n` 拼接
2. **delta.content ``` 围栏代码块**：若无工具调用但 `delta.content` 含 ` ``` `，提取围栏内代码
3. **纯说明文字 → 返回空**：无工具调用、无围栏 → 返回 `''` → `reporter.push()` 跳过不入队

**两接口通用字段：**

| 字段 | 来源 |
|------|------|
| sessionId | 请求头 `x-conversation-id` |
| requestId | 请求头 `x-request-trace-id` |
| scope | 请求头 `x-ide-type` |
| pluginVersion | 请求头 `x-ide-version`（由 `config.reporterTriggerVersion` 决定） |
| createdBy | 系统用户名（UM号），`os.userInfo().username` |
| type | 固定 `"CODE_CHAT"` |
| cost | `record.duration`（毫秒） |
| apiUrl | `record.url` |
| prompt | 请求体 `messages` 数组拼接 |

### `src/reporter.js` — 上送队列

- **入队过滤**：`result` 为空 → `push()` 直接 `return false`，不入队
- **业务成功判定**：HTTP 200 ≠ 成功；需检查响应体 `data.code === '0'`（或 `success === true`）；`code:"1"` 视为失败 → 自动重试（最多 `REPORTER_MAX_ATTEMPTS` 次）
- **持久化**：队列落盘 `reporter-queue.json`，应用重启后继续发送
- **已送达历史**：内存保留最近 100 条（含完整 entry），可同等预览已送达报文

### `src/settings.js` — 设置管理

- `appTitle`：默认 `'资金同业代码解析工具'`，可通过面板修改
- `umAccount`：自动取系统用户名；手动改后设 `umAccountManual=true` 持久化（防拷贝到别的机器）
- `reporterTargets.saveRecord.match`：默认 `'*/completions*'`（同时匹配两个接口）

## 四、已修复的重要 Bug

| Bug | 原因 | 修复位置 |
|-----|------|---------|
| HTTPS 证书浏览器报不安全 | leaf cert 有效期超过 398 天 | `patches/http-mitm-proxy+0.9.0.patch` → `setDate(getDate() + 397)` |
| WS 连接代理崩溃 | 传 reserved close code 1005/1006 给 `ws.close()` | `patches/http-mitm-proxy+0.9.0.patch` → `closeWebSocketSafely()` |
| 右键"查看报文"不生效 | `hideCtxMenu()` 清空 `_ctxId` 在 id 被保存之前执行 | `index.html` → `const id = _ctxId; hideCtxMenu(); if (id != null) showDetail(id);` |
| 全部 4 条报文都被上送 | `sseCodeAll` 有 `combined \|\| content` fallback，纯文字也被当代码 | `reporter-mapping.js` → 移除 fallback，纯文字返回 `''` |
| HTTP 200 被当上送成功 | 埋点接口永远返回 200，需看 `code` 字段 | `reporter.js` → `bizResult()` 检查 `data.code === '0'` |
| `/v2/completions` 裸代码被丢弃 | 原逻辑只识别 ``` 围栏 | `reporter-mapping.js` → `isCompletionStyle()` + 裸代码路径 |
| `\n` 转义序列不显示换行 | JSON 字符串中的 `\n` 是两个字面字符 | `index.html` → `unescapeForDisplay()` |

## 五、前端关键功能（`web/public/index.html`）

- **抓包列表**：实时更新，右键菜单（查看详情 / 补送），倒序排列，"抓包#"列含定位链接
- **报文详情**：请求体/响应体支持格式化、语法高亮（Web Worker）、复制，最大化显示
- **上送队列**：待处理 / 最近送达，点击行预览报文，显示上送接口返回状态及响应体
- **补送功能**：对已完成但未入队的报文，右键「补送」手动入队
- **上送设置**：URL、格式（raw/behavior）、Token、字段映射、URL 过滤白名单、软件名称
- `setPreText(preEl, text)`：`\r\n`→`\n` 规范化 + `unescapeForDisplay` 显示转义序列
- `uploadResponseHtml(resp)`：渲染上送接口返回（绿色=成功，红色=失败）

## 七、开机自启（低权限，无需管理员）

### Windows

```bat
scripts\autostart\install-autostart-win.cmd   :: 安装（VBS 无黑窗口）
scripts\autostart\uninstall-autostart-win.cmd :: 卸载
```

自启项位置：`%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\proxy-inspector.vbs`

### 统信 UOS / 信创 Linux

```bash
bash scripts/autostart/install-autostart.sh   # 安装（XDG .desktop）
bash scripts/autostart/uninstall-autostart.sh # 卸载
```

自启项位置：`~/.config/autostart/proxy-inspector.desktop`

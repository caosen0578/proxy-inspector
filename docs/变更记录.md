### 上送字段与口径（详见 [CodeBuddy代码提取规则.md](CodeBuddy代码提取规则.md)、[字段映射分析.md](字段映射分析.md)）
- **补齐 `acceptCodeLines` / `acceptCodeSize`**：接口文档标"否"但后端按必填校验；按 `acceptResult` 计数（`field/lineCount`、`field/byteSize`）。
- **修复 `result` 缺代码**：`sseContent` → **`sseFullReply`**（完整回复 = 旁白 + 工具写入文件的围栏块）。
  根因：Agent 模式经 `tool_calls`（write_to_file/replace_in_file）写码，`delta.content` 只有旁白 → 旧 `result` 丢代码；
  更严重的是**纯工具轮 `result=''` 被内容过滤整轮漏送**（"代码应用到文件却没上送"的主因），修复后恢复。
- **修复 `extractGeneratedCode` 短路**：改为「工具代码 ∪ 旁白围栏代码」并按文本去重，不再丢同轮旁白里贴的代码。
- **`sseFullReply` v2 — result 全量工具留痕**：本轮所有工具调用按流式顺序写入 result——写码工具为
  `[工具名] 文件 路径：`+围栏块（不截断），其余工具为 `> 调用 名: {参数摘要}` 单行（截断），便于排查时序。
- **映射自动迁移**：`settings.js` `load()` 把旧 `result=sseContent` 升级为 `sseFullReply`
  （已分发机器 settings.json 覆盖默认值的运行时兜底；教训：**改默认映射必须配迁移 + 同步 env 文件**）。
- **固有边界（不可修，详见 [待解决问题.md](待解决问题.md)）**：`replace_in_file` 只有 `new_str` 改动块、
  整文件在 IDE 本地不经网络；`execute_command` shell 写文件不计——被动抓包物理边界。

### 版本治理（配合后端 user-behavior-track 的 `/app-version/latest`，详见 [使用手册.md](使用手册.md)）
- **新版本通知**：`updateCheckUrl` 检查地址；`GET /api/update/check`（后端拉取+10min缓存+fail-open+每2h后台自查）；
  页头蓝色横幅 + 「检查更新」按钮 + 按版本「不再提醒」。版本比较取版本串里的点分数字（末位累加即可）。
- **强制停用旧版**：后端返回 `minVersion`，本地低于它 → 代理对新请求回 503（**不影响已入队上送**、面板照常），红色不可关横幅。
- **停用倒计时**：后端返回 `disableAt`（**绝对时刻**，年月日时分秒/本机时区）→ 未到点橙色倒计时横幅、到点自动停用；
  用绝对时刻故**不受代理重启影响**，`isDisabled()` 每请求实时判断。
- **检查间隔后台可配**：后端返回 `checkIntervalSeconds`（后台页可按 时/分/秒 填）→ 前端面板轮询间隔按它走
  （默认 2h、下限 10 秒，自重排 setTimeout）。后端自查首次由 5 秒改 **2 秒**。
- **横幅常驻**：有新版即常驻显示（刷新仍在、升级自动消失），移除「本版本不再提醒」。
- **后端变更及时反映**：`check()` 外呼缓存 TTL + 后端自查间隔改为跟随 `checkIntervalSeconds`（`intervalMs`，下限10秒/默认2h）——
  后台延长/撤销 `disableAt`、改 `minVersion/notes` 等，客户端在一个间隔内自动切横幅状态（红↔橙↔蓝），手动检查即时。
- **横幅文案**：notes 自动拼「更新说明：」前缀；停用/升级句尾用「！」。

### 2026-07-11 计数口径：计数字段与文本字段一一对应

### 2026-07-11 修复 acceptCodeLines/acceptCodeSize 上送恒 0
- 根因：settings.json 里 `field` 源丢 `path`（早期 UI 不认识 field 源、映射页保存时吞掉）→ 计数取 undefined→0，
  实测新批 27/27 条中招。
- 修复：① `settings.js` load() 自愈迁移（field 缺 path 补 `acceptResult`、字段整个缺失按默认补）；
  ② `toSaveRecord` 改**两遍解析**（非 field 先解析、field 后解析）——field 引用与映射键顺序彻底无关；
  ③ 调试 JSONL 增加 codeLines/codeSize/acceptCodeLines/acceptCodeSize 四列（与上送同值，正是靠它发现的）。
- 数据影响：此前上送的记录 acceptCodeLines/acceptCodeSize 为 0，如后端统计需要可用「补送历史报文」重送。

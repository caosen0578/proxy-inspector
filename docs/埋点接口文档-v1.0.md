# 用户行为埋点接口文档

- **版本**：v1.0
- **状态**：正式发布
- **最后更新**：2026-05-20
- **维护团队**：ai 埋点工具组

## 1. 概述

本文档定义了 AI 工具向 AI 埋点端上报用户 AI 交互行为数据的相关接口。主要包含两个场景：

1. **行为记录保存**：在 AI 请求完成并返回结果后，记录完整的交互上下文、耗时、模型信息等。
2. **行为记录更新**：在用户对 AI 生成的结果执行后续操作（如复制、采纳代码等）后，更新相应的行为状态及采纳数据。

### 1.1 环境配置

| 环境类型 | 域名 (Base URL) | 说明 |
|----------|-----------------|------|
| 测试环境 | `http://appdes.fat001.qa.pab.com.cn` | 用于开发联调与功能测试 |
| 生产环境 | `http://appdes.fat.qa.pab.com.cn` | 正式生产环境 |

### 1.2 鉴权方式

所有接口均需在 HTTP Header 中携带 `apiToken` 进行鉴权。

- **Header 参数**：`apiToken`
- **Token 值**：向 `ex-pengqisheng835` 申请

## 2. 接口详情

### 2.1 新增行为记录

- **描述**：插件调用 AI 接口获得响应后，保存本次交互的完整数据。此接口为幂等接口，建议通过 `requestId` 唯一标识一次交互。
- **接口地址**：`/api/userBehavior/saveRecord`
- **请求方式**：POST
- **Content-Type**：application/json

### 2.2 更新行为记录（用户操作反馈）

- **描述**：当用户对 AI 生成的结果执行了后续操作（如复制代码、采纳代码片段、编辑代码等）后，调用此接口更新行为记录中的采纳数据。
- **接口地址**：`/api/userBehavior/updateRecordForAccept`
- **请求方式**：POST
- **Content-Type**：application/json

#### 2.2.1 参数详解

| 字段名 | 类型 | 必填 | 示例值 | 描述 |
|--------|------|------|--------|------|
| requestId | String | 是 | `"78ee7d17-..."` | 关联的原始请求 ID，需与 saveRecord 接口中的 ID 一致 |
| actionType | String | 是 | `"codeCopy"` | 用户操作类型枚举：codeCopy(复制)、codeAccept(采纳)、codeEdit(编辑) 等 |
| acceptResult | String | 是 | `"您好，我是..."` | 操作时的结果快照或变更后的内容 |
| acceptCodeLines | Integer | 是 | `10` | 操作涉及的代码行数 |
| acceptCodeSize | Integer | 是 | `20` | 操作涉及的代码大小（字节） |

#### 2.2.2 响应示例

成功：

```json
{ "code": "0", "msg": "操作成功", "data": null, "success": true }
```

错误：

```json
{ "code": "1", "msg": "requestId不能为空", "data": null, "success": false }
```

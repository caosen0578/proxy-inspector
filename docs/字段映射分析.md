# saveRecord 字段赋值方案（用户行为埋点接口 v1.0）

> 依据 v1.0 文档（2026-05-20）。接口：`POST /api/userBehavior/saveRecord`，鉴权头 `apiToken`。
> 抓包样例：CodeBuddy IDE → `POST /v2/chat/completions`，OpenAI 请求体 + SSE 流式响应（model=dsv4）。
> 报文头一律小写：`x-user-id`、`x-conversation-id`、`x-request-trace-id`、`x-ide-name`、`x-ide-version` 等。

## 一、必填字段赋值方案

| 字段名 | 类型 | 示例值 | 赋值来源 | 说明 |
|--------|------|--------|---------|------|
| pluginVersion | String | `"IDEA_3.1.4.1"` | 配置项 `reporterTriggerVersion` | 默认 `proxy-inspector`，可在设置页改 |
| createdBy | String | `"chenyulan480"` | 系统用户名 `os.userInfo().username` | UM号，右上角可手动改 |
| sessionId | String | `"0dc82829-..."` | 请求头 `x-conversation-id` | |
| requestId | String | `"78ee7d17-..."` | 请求头 `x-request-trace-id` | |
| type | String | `"CODE_CHAT"` | 固定值 | |
| result | String | 代码文本 | SSE `sseCodeAll`（见代码提取规则） | 只取代码，不含纯文字 |
| acceptResult | String | 代码文本 | 同 result | 抓包阶段无法区分是否采纳，先填模型产出代码 |
| prompt | String | `"你是谁"` | 请求体 `messages` 拼接 | 含 system/user 全部 |
| scope | String | `"RooCode"` | 请求头 `x-ide-type` | 如 `CodeBuddyIDE` |
| isStatistics | Integer | `1` | 固定值 `1` | |
| modelName | String | `"dsv4"` | SSE `model` 字段 | 取响应中的 model |
| promptTokens | Integer | `1000` | SSE 末 chunk `usage.prompt_tokens` | |
| completionTokens | Integer | `100` | SSE 末 chunk `usage.completion_tokens` | |
| totalTokens | Integer | `10100` | SSE 末 chunk `usage.total_tokens` | |

## 二、选填字段赋值方案

| 字段名 | 类型 | 赋值来源 | 说明 |
|--------|------|---------|------|
| cost | Integer | `record.duration`（毫秒） | 代理测得响应耗时 |
| apiStatusCode | String | HTTP 状态码字符串 `"200"` | AI 业务码在 SSE 体里没有 |
| clientResponseCode | String | `"200"` | 同上 |
| triggerType | String | 固定 `"auto"` | |
| apiUrl | String | `record.url` | |
| finishReason | String | SSE 末 chunk `choices[0].finish_reason` | |
| codeLines | Integer | `result.split('\n').length` | 代码行数 |
| codeSize | Integer | `Buffer.byteLength(result, 'utf8')` | 代码字节数 |
| language | String | 请求体 `extra.language` | 补全接口有，对话接口无 |
| filePath | String | 请求体 `extra.file_name` | |
| repository | String | 请求体 `extra.repo_name` | |

## 三、抓包无法获取的字段（保持 null）

`language`（对话接口）、`batchNo`、`commandType`、`acceptCodeLines`、`acceptCodeSize`、
`resultId`、`templateUuid`、`modelScope`、`instructionPath`、`knowledgeUuid`、
`usage`、`isUseCache`、`waitCost`、`requestWaitCost`、`promptMd5`、`promptSize`

## 四、updateRecordForAccept（用户操作反馈）

> 此接口需要 IDE 端操作事件，**抓包无法实现**，仅作记录。

| 字段名 | 类型 | 必填 | 能否抓包取 |
|--------|------|------|-----------|
| requestId | String | 是 | ⚠️ 需与 saveRecord 一致 |
| actionType | String | 是 | ❌ 用户操作事件（codeCopy/codeAccept/codeEdit） |
| acceptResult | String | 是 | ❌ |
| acceptCodeLines | Integer | 是 | ❌ |
| acceptCodeSize | Integer | 是 | ❌ |

## 五、请求头字段参考（CodeBuddy 4.2.x）

| 报文头 | 值示例 | 对应字段 |
|--------|--------|---------|
| `x-ide-type` | `CodeBuddyIDE` | scope |
| `x-ide-version` / `x-product-version` | `4.2.5` | pluginVersion |
| `x-user-id` | UUID | 参考值（非 UM 号） |
| `x-conversation-id` | UUID | sessionId |
| `x-request-trace-id` | UUID | requestId |
| `x-agent-intent` | `craft` / `CodeCompletion` | 接口类型判断 |

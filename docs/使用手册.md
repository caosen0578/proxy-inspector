# Proxy Inspector 使用手册

HTTP/HTTPS 抓包代理工具：抓取 AI 接口报文，映射成「用户行为埋点接口」上送到埋点平台，统计个人代码生成率。

---

## 一、打包（分发给别人用）

> 适用：把工具发给同事/部署到内网无 npm 源的机器。

1. 在**本机**（已能正常运行的机器）双击运行 **`pack.cmd`**
2. 脚本会自动：`npm install`（含证书补丁）→ 复制运行所需文件到 `dist\proxy-inspector\`
3. 把 `dist\proxy-inspector` 整个文件夹**压缩成 zip** 发给对方

**打包会自动排除**（不会带上你的私人数据）：
- `certs/`（含本机 CA 私钥，每台机器要生成自己的）
- `settings.json`（你的 apiToken、UM 号等个人配置）
- `reporter-queue.json`（你抓到的报文队列）
- 临时/文档文件

> ⚠️ 不要手动整个目录拷贝分发，否则会带上 `settings.json`，导致对方机器显示你的 UM 号、用你的 token。务必用 `pack.cmd`。

### 发布新版本（更新通知）

工具带「新版本通知」：管理员在「上送设置 → 新版本检查地址」填内网检查地址后，
每台机器的面板会**定时自动检查**（默认每 2 小时，可由后端 `checkIntervalSeconds` 字段配置），
有新版在页头下方**常驻**弹蓝色横幅（含下载链接；只要版本不同就一直显示、刷新后仍在，升级后自动消失）；
页头「检查更新」按钮可随时手动查（会提示"已是最新"或检查失败原因）。

检查地址是**后端接口或静态 version.json 均可**，只要返回一条 JSON 数据（后端配什么就返回什么）：
```json
{ "version": "1.1.0",
  "downloadUrl": "http://内网服务器/proxy-inspector/proxy-inspector-1.1.0.zip",
  "notes": "一句话更新说明（会显示在横幅上）",
  "minVersion": "1.0.5",
  "disableAt": "2026-07-15 00:00:00",
  "checkIntervalSeconds": 1800 }
```

> `checkIntervalSeconds`（可选）：面板版本检查的轮询间隔（**秒**），空/不填=默认 7200（2 小时），客户端夹取下限 10 秒。
> 后端「版本管理」页可按 **时/分/秒** 选择填写（下发到接口即换算成秒）。想让通知/停用更快触达可调小
> （客户端后端有 10 分钟外呼缓存兜底，不会打爆检查服务）。

**强制停用旧版本（`minVersion` + `disableAt`，都可选）**：通知很久用户仍不升级时，管控端在返回数据里加
`minVersion`（最低可用版本），可再配 `disableAt`（**绝对停用时刻**）给一个宽限倒计时：

| 后端下发 | 本地版本低于 minVersion 时的表现 |
|---|---|
| 仅 `minVersion` | **立即停用**：红色不可关闭横幅 + 代理拦新请求回 503 |
| `minVersion` + 未来的 `disableAt` | **橙色倒计时横幅**「将于 X 停用，剩 N 天 N 时 N 分」，到点自动转为停用 |
| `minVersion` + 已过的 `disableAt` | 等同立即停用 |

停用后：
- **代理暂停转发新请求**（IDE 的 AI 请求收到 503 提示升级）——倒逼升级；
- **不影响已抓到的数据**：上送队列照常排空、面板照常可访问、抓包记录不丢。

关键设计与注意：
- `disableAt` 填**年月日时分秒**（如 `2026-07-15 00:00:00`，按运行机器所在时区解读；秒可省，
  也接受 `2026/07/15 00:00`）。是**绝对时刻**、不要用"多少天后"这种相对值——那样每次请求都刷新，
  代理一重启倒计时就归零。用绝对时刻，客户端每次拿"当前时间 vs disableAt"算，**倒计时不受重启影响**，
  到点即停（后台每次代理请求实时判断，不必等下次检查）。
- 检查服务临时故障**不误伤**（fail-open：拉不到结果维持上一次状态，从未成功过=不管控）；
  后台每 2 小时自查、**不依赖用户开着面板**。升级或管控端撤销 `minVersion` 后自动恢复。

发版流程（分发者）：
1. **改「上送设置」里的版本号**（即页头显示的那个，如 `fat001- v1.0.2` → `fat001- v1.0.3`）——
   版本比较取其中的数字段（`1.0.3`），末位一直累加即可（`1.0.9 → 1.0.10` 也能正确比较）；
   接口返回的 `version` 同理可带前缀（`- v1.0.3` / `1.0.3` 均可）
2. `pack.cmd` 打包 → zip 传到内网下载位置
3. 让检查地址返回新的 `version` / `downloadUrl` / `notes`（后端接口改数据，或静态文件改内容）
4. 用户收到横幅 → 点下载 → 解压覆盖（保留自己的 `settings.json`/`certs/` 不受影响，
   因为 zip 里本就不含这些）→ 重启工具

> 联调：`D:\code\behavior-validator` 提供模拟接口 `GET /api/version/latest`
> （query 参数可覆盖 version/downloadUrl/notes，见其 README「用法三」）。

## 二、启动（运行工具）

> 前提：机器已安装 **Node.js ≥ 20**（命令行执行 `node -v` 查看）。
> ⚠️ **必须 Node 20 及以上**：低版本无 Happy Eyeballs，MITM 内部回环隧道会连 IPv6 `::1`（内部服务器只绑 `127.0.0.1`）导致 HTTPS 拦截失败，客户端报 3003 `Cannot connect to API: Headers Timeout`、后台无日志。patches 已强制内部隧道走 `127.0.0.1` 兜底，但仍建议升级 Node。

**方式 A：双击 `start.cmd`**（推荐，最简单）
- 自动设置端口并启动，弹出黑色命令行窗口
- 启动成功后会显示彩色提示和访问地址
- **改端口**：用记事本打开 `start.cmd`，改顶部两行 `set PROXY_PORT=` / `set UI_PORT=` 即可

**方式 B：命令行**
```bat
cd proxy-inspector
npm install   :: 仅第一次需要（离线包已自带依赖，可跳过）
npm start
```

**方式 C：麒麟 / 信创 Linux（终端运行）**

> 麒麟等信创系统的文件管理器**双击 `.sh` 默认不会执行**（安全策略），必须在**终端**里运行。
> 前提：已安装**与本机 CPU 架构匹配**的 Node.js（鲲鹏/飞腾为 `aarch64`、兆芯/海光为 `x86_64`；终端执行 `node -v` 能看到版本即可）。

1. 打开**终端**，`cd` 到解压后的目录：
   ```bash
   cd ~/下载/proxy-inspector      # 换成你实际的解压路径
   ```
2. **首次**赋予执行权限（只需一次）：
   ```bash
   chmod +x start.sh
   ```
3. 启动：
   ```bash
   bash start.sh
   # 或改端口：PROXY_PORT=28899 UI_PORT=28900 bash start.sh
   ```
4. **保持这个终端窗口开着**即为运行中；退出按 **Ctrl+C**（会先持久化上送队列再退出）。

> 若 `bash start.sh` 报 `$'\r': command not found` 之类错误，是脚本被 Windows 改成了 CRLF 换行，执行一次转换即可：`sed -i 's/\r$//' start.sh`。

启动后：
- **代理端口 28899** ← 把浏览器/系统/IDE 的代理设为 `127.0.0.1:28899`
- **界面 28900** ← 浏览器打开 `http://127.0.0.1:28900`

### 端口被占用怎么办
启动时若端口被占用，命令行会提示占用进程的 **PID 和名称**，并询问：
```
是否结束该进程 (PID xxxx) 并继续启动？ [y/N]
```
输入 `y` 回车即结束占用进程并继续启动；输入 `n` 取消。
（也可改 `start.cmd` 顶部端口避开冲突。）

## 三、HTTPS 抓包：安装根证书

抓 HTTPS 必须信任根证书，否则浏览器/IDE 报「不安全」。

1. 打开界面 `http://127.0.0.1:28900`，先访问一次任意 HTTPS 网站（让证书生成）
2. 点界面顶部横幅或右上角 **「⬇ 下载根证书」**（Windows 用 **`.cer`**）
3. **双击 `.cer`** → 安装证书 → **本地计算机** → 「将所有证书放入下列存储」→ 选 **「受信任的根证书颁发机构」** → 完成
4. **重启浏览器/IDE**

> Windows 不认 `.pem` 双击安装，务必用 `.cer`。

**麒麟 / 信创 Linux 安装证书**：改点 **「⬇ 下载根证书」** 里的 **`.pem`**，然后二选一——
- **导入浏览器**（推荐，最省事）：麒麟自带浏览器 / 火狐 → 设置 → 隐私与安全 → 证书 → 「查看证书」→ 「证书颁发机构」标签 → **导入** `ca.pem` → 勾选「信任由此证书颁发机构来标识网站」→ 确定，重启浏览器。
- **导入系统信任库**（需管理员）：`sudo cp ca.pem /usr/local/share/ca-certificates/proxy-inspector.crt && sudo update-ca-certificates`（银河麒麟 V10 适用；部分信创发行版命令略有差异）。

## 四、配置上送（埋点平台）

1. 界面右侧点 **「上送设置」** 标签
2. **上送格式** 选 `behavior`（映射为埋点 saveRecord 接口）
3. 填 **埋点平台 BaseURL（含 `/api`）**（如 `http://appdes.fat.qa.pab.com.cn/api`）
4. 填 **apiToken**（向埋点工具组申请）
5. **URL 过滤白名单** 填 `*/completions*`（只上送 AI 补全/对话接口）
6. 点 **保存设置**
7. 用**右上角「上送 开/关」**开关一键开启上送（或在本页开关）

实际上送地址 = BaseURL + `/userBehavior/saveRecord`，鉴权头 `apiToken`。
**注意**：`/api`（各环境网关前缀）写在 BaseURL 里，如 `http://xxx.com.cn/api` → 实际打到 `http://xxx.com.cn/api/userBehavior/saveRecord`。

### 上送超时（仅管理员）
「上送设置」底部 **「上送超时（毫秒）」**：单次上送等待后端响应的最长时间，默认 **15000**，范围 1000~120000。
- **务必大于上送后端的最坏响应时间**。若设得太小，后端「慢成功」时客户端会先超时、误判失败而**重发，造成重复上送**。
- 和其它上送设置一样，**仅管理员**可改（非管理员保存会被拒）。

### UM 号（= createdBy，统计个人代码生成率）
- 右上角 **「UM xxx」** 自动取当前机器系统用户名
- 不对就点一下手动改，改后会记住（每台机器各自独立）

## 五、日常使用

| 功能 | 位置 | 说明 |
|------|------|------|
| 看流量 | 左侧列表 | 实时显示抓到的请求，可按 URL/方法/状态/**类型**筛选 |
| 看报文详情 | 点左侧某条 | 请求/响应头、Body，**可一键复制** |
| **SSE 解析** | 响应 Body 区「⊞ SSE 格式解析」 | 流式响应解析出正文/思考链/tokens |
| 配字段映射 | 「字段映射」标签 | 两个上送接口分开配，字段来自接口文档 |
| 看上送队列 | 「上送队列」标签 | 待送/失败统计，点条目预览上送报文 |
| **补送漏送的** | 「上送队列」→「补送历史报文」 | 上送开晚了？勾选历史报文批量/单条补送 |
| 加规则 | 「规则管理」标签 | mock/改头/拦截（与上送独立，按需用） |

### 上送失败重试
失败自动重试最多 **5 次**，达上限转「失败」终态、停止自动重试。需要时在「上送队列」点 **「重试失败项」** 手动重试。

> 注：自动重试是「至少一次」投递——若后端其实已成功、只是回包慢/丢了，重试会让后端**重复收到**同一条。后端（中转服务）按 `requestId` 做了接收端幂等去重，重复不产生脏数据；想从源头少重发，把上面的「上送超时」调到大于后端最坏响应时间即可。

### 退出
命令行窗口按 **Ctrl+C**，会先持久化未完成的上送队列再退出（不丢数据）。重启后自动恢复继续送。

## 六、常见问题

**Q：换了机器，UM 号显示的还是别人的名字？**
A：你可能手动整目录拷贝带上了 `settings.json`。删掉它重启即按本机重新探测；或点右上角 UM 手动改。今后用 `pack.cmd` 打包可避免。

**Q：代码补全（行内 ghost text）开了代理就不回写？**
A：补全是延迟敏感接口。工具已对 `*/completions*` 启用「低延迟直通」（不强制 gzip、关 Nagle、强制连接关闭）尽量保通。可在「上送设置」底部「低延迟直通 URL」调整。若仍不行，是 IDE 与 MITM 代理的兼容问题——**注意：统计数据采集不受影响**（报文照样抓到、照样上送）。

**Q：有些接口漏上送了？**
A：「上送队列」→「补送历史报文」，勾选补送。前提是抓包列表里还在（默认保留最近 500 条）。

**Q：上送的 result 是 SSE 原文还是解析后的？**
A：解析后的纯文本。`result` 用 `sseFullReply`：模型旁白 + 本轮**所有工具调用按顺序留痕**——写代码的工具（write_to_file/replace_in_file）以 ``` 围栏代码块嵌入（标 `[工具名]` 和文件路径，不截断），其余工具（read_file 等）以 `> 调用 工具名: {参数摘要}` 单行记录。`acceptResult` 用 `sseCodeAll` 只取其中代码。两者都已剥离 `data:`/`[DONE]`/思考链。

**Q：怎么排查某条上送的内容对不对？**
A：管理员在「上送设置」开启「调试落盘(JSONL)」，之后每条要上送的记录会在工具目录生成 `upload-debug-log.jsonl`（每行一条 JSON：time/id/requestId/prompt/result/acceptResult/requestRaw/responseRaw，`id` 与抓包页对应）。面板有「下载 JSONL」按钮，也可直接本地 grep/jq。⚠️ 内容含代码明文，排查完请关闭开关并删除文件。

## 七、目录说明

```
proxy-inspector/
├── start.cmd          # Windows 启动（改端口在这）
├── start.sh           # Linux/麒麟 启动（终端 bash start.sh）
├── pack.cmd           # 打包分发
├── src/               # 后端代码
├── web/               # 界面
├── mapping-presets/   # 字段映射预设
├── patches/           # 证书有效期补丁（勿删）
├── certs/             # 自动生成的本机证书（勿分发）
├── settings.json      # 个人配置（勿分发）
└── reporter-queue.json# 上送队列（自动持久化）
```
